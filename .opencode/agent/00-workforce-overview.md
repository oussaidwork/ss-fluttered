# 00 — Workforce Overview

## Purpose

This file defines the full workforce model for high-quality fullstack development.

## Roles

| Role | Main responsibility |
|---|---|
| Product Architect | Translate business goals into architecture |
| Fullstack Engineer | Build end-to-end features |
| Frontend Engineer | Implement UI and interaction quality |
| Backend Engineer | Build APIs and server logic |
| Database Engineer | Design schema and data integrity |
| DevOps / SRE | Own CI/CD, infra, and operations |
| QA Test Engineer | Validate correctness and regression safety |
| Security Engineer | Protect the system from threats |
| UX / Accessibility Engineer | Ensure usability and a11y |
| Tech Lead Reviewer | Enforce architecture and maintainability |
| Release Manager | Ship safely and rollback if needed |

## Workflow phases

1. Discover
2. Design
3. Build
4. Verify
5. Review
6. Release
7. Monitor
8. Improve

## Handoff model

Every role must produce a handoff packet containing:

- Summary
- Decisions made
- Risks
- Open questions
- Next steps
- Artifacts produced

## Quality gates

- DOR
- DOD
- CI pass
- PR review
- Security scan
- Accessibility check
- Release checklist

## Anti-patterns

- Skipping handoff
- Merging without tests
- Releasing without rollback
- Ignoring observability
- Treating UX as decoration
- Treating security as an afterthought

## Success metric

A feature is successful only when it is:
- Correct
- Maintainable
- Secure
- Accessible
- Observable
- Releasable
