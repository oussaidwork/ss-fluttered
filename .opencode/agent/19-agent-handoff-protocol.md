# 19 — Agent Handoff Protocol

## Purpose

Ensure every handoff between agents is clear, complete, and actionable.

## Handoff packet

Every handoff must include:

1. Summary
2. Decisions made
3. Risks
4. Open questions
5. Next steps
6. Artifacts produced
7. Review status
8. Required follow-up

## Handoff template

```md
## Summary
<short description>

## Decisions
- <decision>

## Risks
- <risk>

## Open questions
- <question>

## Next steps
- <step>

## Artifacts
- <file/link>

## Review status
- <status>

## Follow-up required
- <task>
```

## Handoff rule

If any section is missing, the receiving agent must ask for it before proceeding.

## Best practice

Keep the handoff concise but complete. Do not hide assumptions.
