# Fullstack Agent Workforce

This skill defines a full workforce for building fullstack software with strong quality controls.

## When to use

Use this workforce when:

- A project needs a clear engineering process
- Multiple agents must collaborate on the same feature
- You want to prevent quality regressions
- You need a repeatable review and release flow
- You want to standardize handoffs between roles

## Core principle

Every agent must produce a clear handoff packet before the next agent starts.

## Primary workflow

1. Product architect defines the problem and architecture
2. Fullstack engineer implements the feature
3. QA, security, and UX agents validate it
4. Tech lead reviews the final shape
5. Release manager ships it
6. SRE monitors it

## Quality gates

- Definition of Ready
- Definition of Done
- Code review
- Test coverage
- Security scan
- Accessibility check
- Observability check
- Release checklist

## Required artifacts

- PRD
- ADR
- Test plan
- PR
- Release notes
- Runbook
- Postmortem

## Handoff rule

If a handoff packet is missing, the next agent must ask for it before proceeding.
