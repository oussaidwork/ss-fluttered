# 13 — Definition of Ready / Done

## Definition of Ready

A task is ready when:

- The problem is clear
- The user value is clear
- Acceptance criteria are explicit
- UX flows are available
- API contracts are drafted
- Edge cases are listed
- Risks are documented
- Test strategy is drafted

## Definition of Done

A task is done when:

- Code is implemented
- Tests are added
- Docs are updated
- Lint passes
- Typecheck passes
- Build passes
- Security review is complete
- Accessibility review is complete
- Observability is ready
- Rollback plan is ready

## Ready checklist

- [ ] Problem statement
- [ ] Acceptance criteria
- [ ] UX flow
- [ ] API contract
- [ ] Risk list
- [ ] Test plan

## Done checklist

- [ ] Implementation complete
- [ ] Tests complete
- [ ] Docs complete
- [ ] Review complete
- [ ] Security complete
- [ ] Accessibility complete
- [ ] Observability complete
- [ ] Rollback ready

## Rule

If a task is not ready, it should not be estimated or implemented.
